# Brightline HR Analytics

An end-to-end people-analytics project: synthetic data generation → exploratory
analysis → a logistic regression flight-risk model → interactive dashboard. Built
to demonstrate the kind of HR data work I do day-to-day (attendance/leave
tracking, workforce reporting, HR dashboards) in a form that's shareable
outside my employer's systems.

**[Read the analysis notebook](notebooks/exploratory_analysis.ipynb)** · Run the
dashboard locally with the instructions below.

## Why this project

In my HR Operations role I build attendance trackers, KPI dashboards, and
workforce reports for a multi-branch retail business - but that work lives in
internal, confidential spreadsheets I can't share publicly. This project rebuilds
the same kind of analysis - headcount trends, attrition drivers, tenure patterns -
end to end, in the open, on a synthetic dataset.

**The business question it answers:** where is a retail chain losing employees,
and why - so HR can target retention effort instead of applying blanket policies.

## Important: the data is synthetic

Every employee record here is generated by [`data/generate_data.py`](data/generate_data.py)
using `Faker` plus a seeded, rule-based attrition model. **It does not contain,
derive from, or resemble any real employer's data.** It exists to give the
analysis something realistic to work with - department-level turnover gaps,
early-tenure flight risk, performance/attendance signals - while keeping the
repository fully shareable.

900 employees · 7 branches · 8 departments · January 2022 – June 2026

## What's in here

| File | What it does |
|---|---|
| `data/generate_data.py` | Generates the synthetic employee dataset (seeded, reproducible) |
| `src/analytics.py` | Reusable analytics functions (attrition rate, tenure, headcount-by-month, etc.) shared by the notebook and the dashboard |
| `src/risk_model.py` | Logistic regression "flight risk" model - trains, evaluates, and scores active employees |
| `notebooks/exploratory_analysis.ipynb` | The analysis walkthrough - charts, model evaluation, findings, recommendations |
| `app.py` | Interactive Streamlit dashboard with live filters and a flight-risk worklist |

## Key findings

1. **Sales & Retail Ops and Warehouse & Logistics** carry the highest attrition,
   and the effect survives controlling for performance, attendance, and tenure
   in the regression - it's a real structural risk, not just a side-effect of
   who staffs those departments.
2. **Entry-level attrition is roughly double Senior-level attrition.**
3. **A majority of leavers exit within 6 months of joining** - tenure-to-date
   is the strongest single predictor in the model, pointing at an
   onboarding/early-engagement problem more than a long-tenure one.
4. **Lower performance ratings and lower leave balances both predict higher
   risk**, consistent with disengagement showing up before a formal exit.
5. **"Better opportunity" is the single largest stated exit reason**, which
   usually signals a pay or career-path gap rather than a culture problem.

A logistic regression trained on the same data reaches a held-out **ROC-AUC of
0.92** and is used to score every currently-active employee with a flight-risk
probability - the dashboard's "Predicted flight risk" section turns that into
a ranked worklist HR could actually act on.

Full reasoning, the model evaluation, and methodology caveats (it's a
point-in-time classifier, not a survival model - see the notebook for why
that distinction matters) are in the
[notebook](notebooks/exploratory_analysis.ipynb).

## Running it locally

```bash
git clone https://github.com/<your-username>/brightline-hr-analytics.git
cd brightline-hr-analytics
pip install -r requirements.txt

# (optional - a pre-generated dataset is already included in data/)
python data/generate_data.py

# Launch the interactive dashboard
streamlit run app.py
```

To regenerate the notebook outputs after changing the analysis:

```bash
jupyter nbconvert --to notebook --execute --inplace notebooks/exploratory_analysis.ipynb
```

## Possible next steps

- Replace the point-in-time logistic regression with a proper **survival
  model** (Cox proportional hazards) to handle right-censoring correctly and
  estimate *when*, not just *whether*, someone is likely to leave
- Add a "what-if" control to the dashboard - adjust an employee's attendance
  or performance rating and see their predicted risk update live
- Deploy the dashboard on Streamlit Community Cloud for a live demo link
